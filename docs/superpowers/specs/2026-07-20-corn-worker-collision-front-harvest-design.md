# 玉米工人碰撞与正面收割设计

## 背景

玉米区工人当前使用独立的 `CornWorker` 状态机，但生成工人时会禁用模板上的物理组件，并且收割站位始终使用固定的 Z 轴负向偏移。工人在田垄末端折返后仍会前往玉米同一侧，因此可能穿过玉米模型，也可能斜向寻找目标。

森林区工人节点包含 `RigidBody` 和 `CapsuleCollider`，移动状态在进入砍伐距离后立即停止并砍面前的树。玉米区需要复制这种外部行为，但继续使用玉米区自己的脚本，不能依赖或修改 `Woodcutter`。

## 目标行为

- 三名玉米工人各自负责一条完整田垄，互不换垄。
- 工人只沿当前田垄的直线方向移动。
- 当前玉米必须始终位于工人行进方向的正前方。
- 工人到达玉米前方的收割站位后立即停止，不能越过玉米中心。
- 工人砍完当前玉米后继续沿原方向处理下一株。
- 到达田垄末端后转身折返，收割站位随行进方向同步翻转。
- 玉米工人具有与森林工人相同规格的 `RigidBody` 和 `CapsuleCollider` 配置。
- 森林区脚本、森林工人配置和森林区运行逻辑保持不变。

## 实现设计

### 玉米工人移动

`CornWorker` 根据目标列表中相邻玉米的位置计算田垄轴向。移动目标不再是固定的 `offsetVec3`，而是“当前玉米位置减去行进方向乘以站位距离”。这样，无论正向还是折返，工人都会停在当前玉米的来向一侧。

每帧移动距离会限制为“当前位置到收割站位的剩余距离”，防止较大的帧间隔让工人一步越过站位或穿过玉米。达到收割距离后立即进入砍伐状态，不再继续更新位置。

目标选择仍使用现有的田垄有序列表。到达列表端点时仅翻转方向，不搜索其他田垄。朝向使用当前行进方向或当前目标方向，使运行、转身和挥砍方向一致。

### 碰撞组件

`ResourceFieldSystem` 创建玉米工人后，为工人根节点获取或新增独立的 `RigidBody` 与 `CapsuleCollider`，并按森林工人的参数配置：

- 刚体不使用重力；
- 胶囊体半径为 `0.5`；
- 圆柱高度为 `1`；
- 胶囊中心为 `(0, 0.3, 0)`；
- 碰撞体保持森林区使用的触发器模式。

物理组件用于保持玉米工人与森林工人的节点能力一致；“不能穿过玉米”的确定性由移动站位和步长夹紧共同保证，因为触发器本身不会产生物理推开效果。

## 数据流

1. `CornFieldProduction` 按 X 坐标生成三条有序田垄目标列表。
2. `ResourceFieldSystem` 生成三个工人，配置玉米专用脚本和物理组件，并把每条田垄分别交给一名工人。
3. `CornWorker` 从当前索引和方向确定下一株玉米。
4. `CornWorker` 计算当前行进方向上的玉米前方站位，直线移动并在站位处停止。
5. 收割完成后索引前进；到端点则翻转方向并沿原田垄折返。

## 边界情况

- 目标在移动过程中被玩家或拖拉机收割：工人跳过不可用目标，但仍沿本田垄寻找下一株。
- 整条田垄暂时没有成熟玉米：工人保持空闲，不换垄；有玉米重生后继续工作。
- 田垄只有一株玉米：工人在该株前停止收割，不执行无意义的方向切换。
- 帧间隔突然增大：移动步长被剩余距离限制，不越过站位。

## 验证

自动回归测试覆盖：

- 玉米工人脚本不引用 `Woodcutter` 或 `Tree`；
- 每名工人只收到一条完整田垄；
- 正向和折返使用相反的站位方向；
- 单帧移动不会越过当前收割站位；
- 玉米工人生成时启用并配置 `RigidBody` 和 `CapsuleCollider`；
- 森林区 `Woodcutter` 不发生修改。

运行时验收：解锁三名玉米工人，观察每人沿一条直线依次收割；工人到玉米前停止并挥砍，不穿过玉米；到田垄尽头后转身沿原路折返。
