# 农场主题镜头角度设计

## 目标

将游戏的镜头调整为参考图所示的透视斜俯视：镜头从场景右下方向看向左上方向，角色位于画面偏下位置，并让前方道路、工作区域和物件高度关系更易辨识。

## 范围

- 保持竖屏设计分辨率和现有地图节点坐标不变。
- 保持透视相机，不切换为正交相机。
- 保持现有角色移动、相机跟随、边界限制和 UI 逻辑。
- 本次不替换农场地图、美术素材或主角模型。

## 方案

修改 `assets/Scenes/DevScene.scene` 中 Main Camera 的初始位置与朝向，并同步调整其 `CameraController` 的序列化参数：

1. 将跟随偏移从当前的后方平移调整为包含高度和横向分量的右下侧偏移。
2. 将注视点略向前方移动，使角色保持在画面下部，前方经营区域获得更多可视空间。
3. 继续使用 `CameraController.updateLookAt()` 计算运行时朝向，避免角色移动后场景内的初始旋转被覆盖。
4. 保持透视投影、现有视场角和裁剪范围，先以镜头位置与注视点完成构图；如实际预览显示取景不足，再单独微调视场角。

## 组件与数据流

`Main Camera` 上的 `CameraController` 每帧根据 `target + offset` 更新相机位置，并根据 `target + lookAtOffset` 更新朝向。本次仅修改这两个输入及场景内用于编辑器首帧预览的相机变换；玩家、地图和 UI 不接收新的数据，也不改动接口。

## 异常与兼容性

若相机运行时缺少目标节点或 Camera 组件，沿用 `CameraController` 现有的提前返回与错误日志。相机边界限制保持开启，新的偏移值必须处于既有 y 轴边界内，避免跟随时被边界强制截断。

## 验证

1. 在 Cocos Creator 3.8.7 打开 `DevScene.scene`，确认编辑器画面为右下至左上的斜俯视构图。
2. 运行预览并移动角色，确认相机持续保持该观察方向，角色不会遮挡主要前方区域。
3. 检查地图边缘移动、UI 显示、碰撞与角色控制没有回归。
4. 将实际效果与第二张参考图比较：应保留透视缩放和立体高度，不应变为平面俯视。
