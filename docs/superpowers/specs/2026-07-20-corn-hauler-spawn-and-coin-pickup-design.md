# 玉米搬运工出生与金币拾取修复设计

## 背景

玉米区存在两个独立运行时问题：搬运工解锁后出生在地下；玩家进入玉米金币区域后，金币不飞入背包，金额也不增加。

玉米区继续使用独立脚本实现对应功能，不导入或复用森林区的 `HaulerNPC`、`CoinCollector` 或森林库存脚本。行为和数值以森林区为基准。

## 搬运工出生高度

### 原因

左右玉米搬运工解锁点的场景世界高度为 `-0.532`，玩家正确站立高度为 `0.222`。当前 `ResourceFieldSystem.spawnHauler()` 直接把解锁点完整世界坐标赋给搬运工；`CornHauler.onEnable()` 随后保留搬运工自身的错误 Y，因此模型出生在地下。

### 设计

新增玉米区自己的搬运工出生坐标计算：

- X/Z 取当前玉米区搬运工解锁点的世界坐标。
- Y 取真实玩家节点的世界坐标。
- 若玩家节点暂时不可用，才使用解锁点原始 Y，并输出一次明确警告。

`spawnHauler()` 在激活搬运工和启用 `CornHauler` 前设置该坐标。`CornHauler.onEnable()` 继续保持当前出生高度，不再把搬运工拉回地下。

## 玉米金币拾取

### 原因

当前 `CornCoinCollector` 仅依赖运行时创建的碰撞体触发 `onTriggerEnter`，并通过全场景第一个 `CoinBackpack` 自动猜测玩家背包。动态碰撞体漏发进入事件或场景存在搬运工克隆组件时，收集器可能永远不进入收集状态或绑定错误背包。

### 设计

`CornCustomerScheduler` 创建本地金币区域时，显式解析场景中的真实 `Player` 节点及其 `CoinBackpack`，并传给独立 `CornCoinCollector.configure()`。

`CornCoinCollector` 保留与森林区一致的触发碰撞监听，同时增加范围检测兜底：

- 每帧使用金币区域 `BoxCollider` 的中心和尺寸判断真实玩家是否处于区域内。
- 碰撞事件或范围检测任一确认玩家在区域内，即按 `collectInterval = 0.05` 逐枚收集。
- 玩家离开碰撞区且范围检测也为否时停止收集。
- 每枚金币等待入槽缩放动画完成后，从本地 `CornStoragePoint` 移除，再以动画类型 4 加入真实玩家金币背包库存。
- 背包拒绝接收时把金币恢复到本地金币库存，金额不增加。
- 接收成功后金币金额增加 5，与森林区一致。

自动全场景搜索仅作为显式绑定不存在时的兼容回退；搜索时必须选择带 `PlayerController` 的真实玩家节点，不能选择搬运工克隆。

## 测试与验收

- 回归测试锁定搬运工出生 X/Z 来自解锁点、Y 来自玩家。
- 回归测试锁定 `CornCoinCollector.configure()` 接收真实玩家节点和 `CoinBackpack`。
- 回归测试锁定碰撞触发和 `BoxCollider` 范围检测共同驱动收集状态。
- 回归测试锁定成功拾取增加 5，失败恢复金币且不增加金额。
- 运行全部 Node 回归测试、场景 JSON 解析及相关 TypeScript 定向诊断。
- Cocos Creator 目视确认左右玉米区搬运工站在地面上，玩家进入左右金币区域后金币逐枚飞入背包且 UI 金额同步增加。

## 非目标

- 不修改森林区搬运工、金币收集器或森林库存。
- 不改变搬运工路线、背包容量、运输节奏或出售槽位置。
- 不改变顾客金币奖励数量和单枚金币价值。
