# 玉米搬运工随身背包设计

## 背景

玉米搬运工当前通过克隆主角根节点生成。主角运行时创建的玉米背包挂点以及木头、金币等携带物节点可能随克隆一起进入搬运工，导致搬运工刚出现时身上已有物品。

搬运工随后又在自身根节点下创建一个容量为 4 的 `2×2` 存放区。该存放区的位置、层叠方式、容量和装货动画均与主角携带玉米的背包逻辑不同。

## 目标行为

- 玉米搬运工出现时，随身玉米数量必须为 0，身上不能显示继承自主角的玉米、木头或金币。
- 搬运工只使用玉米区独立的随身背包脚本，不直接挂载主角的 `MultiResourceBackpack`，也不复用森林区背包脚本。
- 玉米显示在人物背部骨骼挂点上，而不是挂在人物根节点的固定坐标。
- 玉米按主角背包相同的 `1×1` 单列向上堆叠，每层高度为 `0.2`。
- 搬运工背包容量与主角玉米库存一致，使用 `Number.MAX_SAFE_INTEGER`。
- 最多显示 42 个玉米模型；超过 42 个后继续记录数量，但不继续创建显示模型。
- 装货时，玉米从收集区的世界位置以 `0.32` 秒 `sineOut` 动画移动到背包目标位置。
- 卸货时从栈顶取出显示节点；若取出的是超过显示上限的隐藏库存，则创建一个真实玉米节点用于移动到销售区。
- 装货、携带和卸货过程中库存数量守恒。

## 架构

### CornHaulerBackpack

新增玉米搬运工专用组件 `CornHaulerBackpack`。该组件只负责搬运工的随身玉米库存和显示，不负责寻路或收集区、销售区状态。

组件持有：

- 玉米显示预制件；
- 背部挂点；
- 当前库存数量；
- 可见玉米节点栈；
- 大容量、42 个显示上限和 `0.2` 层高配置。

组件对 `CornHauler` 提供与转移流程兼容的接口：`hasSpace()`、`addResource()`、`removeResource()`、`hasMovableResource()`、`recoverInterruptedTransfers()` 和 `clearStorage()`。

`addResource()` 接收从收集区取出的真实节点。库存未达到 42 个时，将节点重挂到背部挂点并执行与主角一致的飞入动画；库存已超过显示上限时销毁该显示节点，只增加逻辑数量。

`removeResource()` 优先处理隐藏库存。隐藏库存需要卸货时，根据玉米预制件创建一个真实节点，并从背包栈顶世界位置开始向销售区转移；没有隐藏库存时直接弹出最后一个可见节点。

### 搬运工生成清理

`ResourceFieldSystem` 在搬运工仍为非激活状态时清理克隆主角带来的携带物：

- 删除名称以 `ResourceBackpack_` 开头的运行时资源挂点；
- 清空克隆 `backpackMount` 下的木头显示节点；
- 清空克隆 `coinBackpackMount` 下的金币显示节点；
- 不删除人物模型、骨骼、动画或空挂点本身。

清理完成后，以克隆人物的背包挂点作为模板，在相同骨骼父节点下创建 `CornHaulerCarryMount`，复制模板的位置、旋转和缩放。随后创建并清空 `CornHaulerBackpack`，最后才激活搬运工，因此清理过程不会产生一帧的旧物品闪现。

### CornHauler 适配

`CornHauler.carryStorage` 改为 `CornHaulerBackpack`。收集区和销售区仍使用 `CornStoragePoint`，状态机、移动路径和转移节奏保持不变。

装货流程：`CornStoragePoint -> CornHaulerBackpack`。

卸货流程：`CornHaulerBackpack -> CornStoragePoint`。

转移方法只依赖两种存储共有的公开接口，不读取任一组件的内部节点数组。

## 边界情况

- 搬运工生成时主角背包为空或没有某个挂点：清理逻辑安全跳过缺失挂点，并在人物根节点下使用背部默认位置创建玉米挂点。
- 搬运工生成时主角背包已有大量物品：所有克隆显示节点在搬运工激活前被清除，搬运工库存仍从 0 开始。
- 库存超过 42：保持 42 个可见模型，额外库存仅计数；卸货时为隐藏库存补建转移节点。
- 应用切到后台时动画中断：恢复逻辑按背包数量与可见上限重建可见节点状态，不把模板节点或人物模型计入库存。
- 玉米预制件缺失：拒绝接收需要隐藏或补建的库存，并保持来源库存不丢失。

## 验证

自动回归测试覆盖：

- 搬运工激活前执行继承携带物清理；
- 初始数量为 0，挂点无玉米模型；
- 背包为 `1×1` 单列、层高 `0.2`；
- 容量为 `Number.MAX_SAFE_INTEGER`，最多显示 42 个模型；
- 第 1、2、42、43 个玉米的数量和显示节点数量正确；
- 装货动画参数与主角玉米背包一致；
- 隐藏库存卸货时能补建真实节点；
- 装货和卸货后两端总数量不变；
- `CornHauler` 不引用主角或森林区背包脚本。

运行时验收：主角身上携带任意玉米、木头和金币后解锁搬运工，确认搬运工出现时身体为空；搬运工装货后玉米在背部单列向上堆叠，数量可超过 4，并能完整送入玉米销售区。
