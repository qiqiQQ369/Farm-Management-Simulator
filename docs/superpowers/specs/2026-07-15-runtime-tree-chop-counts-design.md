# 运行时树木砍伐次数设计

## 问题

虽然场景中的 `Tree` 组件参数已调整，但 `Tree.onLoad()` 在每次运行时都会强制重写砍伐次数：主角 3 次、伐木 NPC 2 次。因此场景配置不会影响实际玩法。

## 目标

实际运行时统一使用主角 1 刀砍断、伐木 NPC 4 刀砍断，车辆保持 1 刀。

## 方案

仅修改 `assets/_Scripts/Tree.ts` 的 `onLoad()` 初始化覆盖值：

```ts
this.playerChopCount = 1;
this.woodcutterChopCount = 4;
this.vehicleChopCount = 1;
```

保留现有运行时初始化结构，不修改砍伐次数累积、动画时长、木材掉落、树木重生或场景中用户的镜头参数。即使 Cocos 将场景的序列化值保存为其他值，运行时规则也会以这组三种砍伐者次数为准。

## 验证

停止预览后重新运行项目：主角砍任意完整树一次应完成砍伐；伐木 NPC 对任意完整树第四次砍击后才完成；车辆仍一次完成。
