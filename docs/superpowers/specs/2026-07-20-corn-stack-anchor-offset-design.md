# 玉米区堆叠锚点偏移修复设计

## 问题

玉米区顾客结算后生成的金币堆没有居中落在 `CoinPlace` 的白色底板上。玉米产物进入 `Sell1` 出售槽后，整组玉米也与红色槽位中心存在轻微偏移。

两处问题都属于堆叠区域的整体锚点偏移。现有金币排列、玉米排列、层高、旋转和入槽动画不需要改变。

## 目标

- 左右玉米区的金币堆均居中落在各自 `CoinPlace` 白色底板上。
- 左右玉米区的玉米产物均居中落在各自 `Sell1` 红色出售槽中。
- 保持现有堆叠数量、行列、间距、旋转及动画。
- 不修改森林区逻辑，也不影响玉米收集区、玩家背包或搬运工背包。

## 设计

### 金币锚点

`CornCustomerScheduler` 继续在本模块的 `CoinPlace` 下创建独立 `CornCoinDropArea` 和 `CoinStack`。金币区域的整体本地位置对齐 `CoinPlace` 的底板视觉节点中心；`CoinStack` 内部继续使用与森林区一致的 2 列、3 行布局及居中补偿。

左右玉米区使用同一套本地偏移，不读取或修改森林区的 `coinDropArea`。

### 玉米出售槽锚点

`ResourceFieldSystem` 继续为每个 `Sell1` 创建独立 `CornStoragePoint`。只校正出售库存节点的本地位置，使玉米模型的实际视觉中心落在红色槽位中心。现有缩放、旋转、容量、行列、间距和动画保持不变。

该校正只作用于出售槽，不修改 `CornStoragePoint.calculateStackPosition()`，因此不会改变玉米收集区、玩家背包或搬运工背包中的排列。

## 验证

- 增加回归断言，锁定左右 `CoinPlace` 使用相同的本地金币锚点。
- 增加回归断言，锁定左右 `Sell1` 使用相同的出售槽锚点。
- 运行全部 Node 回归测试并验证 `DevScene.scene` 可被 JSON 解析。
- 在 Cocos Creator 中目视确认金币位于白色底板中央、玉米位于红色出售槽中央，且两类入槽动画正常。

## 非目标

- 不改变森林区任何脚本或场景坐标。
- 不改变金币奖励数量和单枚金币价值。
- 不改变玉米产量、库存容量、搬运路线或顾客购买逻辑。
