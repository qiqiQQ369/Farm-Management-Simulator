# Web Release 第二关完整运行链路修复设计

## 背景

项目在 Cocos Creator 本地预览中运行正常，但 Web Mobile Release 构建后，第二关动态创建的对象会出现缺失、位置或动画异常，以及玉米进入售卖槽后顾客不购买的问题。第一关对象主要由场景直接配置，不经过第二关的动态克隆、组件清理和运行时绑定流程，因此没有同样的问题。

已经确认并修复过一类根因：第二关动态对象曾使用 `constructor.name` 筛选组件。Release 压缩会改变 JavaScript 类名，造成正确组件被关闭。该修复已经让玉米收割后能够正常产生玉米产物，也证明正确方向是消除 Release 下不稳定的运行时识别，并在动态对象进入下一阶段前恢复其有效状态。

## 目标

使 Web Mobile Release 中第二关与本地预览表现一致，完整覆盖以下链路：

1. 三名玉米工人的创建、显示、动画、朝向与收割。
2. 玉米拖拉机的创建、显示、碰撞、朝向与收割产物生成。
3. 玉米搬运工的创建、显示、动画、背包、取货与送货。
4. 玉米产物在作物、存放区、搬运工、售卖槽和顾客之间的转移与显示。
5. 玉米顾客识别售卖槽、逐个购买、携带商品和产生金币。

## 不在本次范围内

- 不修改森林区第一关玩法逻辑。
- 不改变本地预览中已经正确的路线、速度、容量、价格、动画节奏或场景布局。
- 不通过关闭 Release 压缩、改为 Debug 构建或复制一套特殊 HTML 构建脚本来规避问题。
- 不修改编辑器自动产生的 `assets/Scenes/DevScene.scene` 用户改动。

## 设计方案

### 1. 稳定的组件识别

检查第二关所有动态创建、组件禁用、组件查找和运行时绑定入口。组件类型必须通过显式导入后的 `instanceof` 或泛型 `getComponent(ComponentType)` 判断，不依赖以下 Release 不稳定方式：

- `component.constructor.name`
- JavaScript 类名字符串
- 运行时被压缩或裁剪后可能变化的反射名称

场景节点业务名称仍可用于定位固定场景锚点，因为节点名称不是 JavaScript 类名；但组件识别不能混用节点名称策略。

### 2. 动态对象激活顺序

第二关角色和产物统一遵循以下顺序：

1. 克隆对象并保持根节点暂时禁用。
2. 清理不需要的旧玩法组件。
3. 添加并绑定玉米区专用组件、路径点、存放区和动画引用。
4. 恢复包含渲染器或动画组件的可视层级。
5. 设置最终世界坐标、旋转和缩放。
6. 启用行为组件、碰撞组件与对象根节点。

这样可以避免 `onEnable` 在绑定未完成时提前执行，也避免 Release 场景序列化保留模板的隐藏子节点。

### 3. 玉米产物转移状态

每次玉米产物从一个容器转移到另一个容器时，都必须保证：

- 节点有效且处于可见层级。
- 不残留上一容器的补间动画或“移动中”状态。
- 重新设置父节点后使用目标容器的本地坐标、旋转和缩放。
- 从源容器移除失败时不增加目标容器数量。
- 进入目标容器成功后，源容器和目标容器的数量与可移动状态保持一致。

这套约束覆盖产物生成、玩家或搬运工取货、搬运工卸货、进入售卖槽和顾客购买。

### 4. 顾客购买链路

玉米顾客调度器只绑定当前玉米模块内的售卖槽、金币区和顾客携带存放点。购买流程依次验证：

1. 当前顾客有效且位于购买点。
2. 当前玉米售卖槽存在可移动产物。
3. 产物从售卖槽移除并成功加入该顾客的携带存放点。
4. 每件成功购买的产物按原有价格生成金币。
5. 买满需求或售卖槽暂时无货时，分别进入离开或等待状态。

Release 下不能依赖第一关组件的类名字符串来寻找第二关售卖槽或顾客背包。

### 5. 回归保护

新增源码级 Release 回归测试，逐项扫描第二关动态链路：

- 不存在 `constructor.name` 或组件类名字符串识别。
- 工人、拖拉机、搬运工在激活前完成绑定并恢复可视层级。
- 所有玉米容器入口都会恢复产物状态。
- 顾客购买明确绑定玉米售卖槽、移动产物并产生金币。

同时运行项目全部 Node 回归测试，并重新生成 `debug: false` 的 Web Mobile Release。构建产物需要确认第二关关键脚本均被打入包内，且浏览器启动没有组件初始化错误。

## 验收标准

- 本地预览原有行为不变。
- Web Mobile Release 中解锁第二关后，三名工人、拖拉机和搬运工均正常出现并运行。
- 玉米被工人、玩家或拖拉机收割时正常产生可见产物。
- 产物能正常进入存放区、搬运工背包和售卖槽，位置与动画保持原有逻辑。
- 顾客会逐个购买玉米，商品显示在顾客身上，并按件产生可拾取金币。
- 森林区第一关全部既有回归测试通过。
- 编辑器生成的 `DevScene.scene` 改动不进入本次提交。
